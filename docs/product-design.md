# Vibe学习记 Product Design

## 1. 产品定位

Vibe学习记是一款面向技术学习者的轻量学习记录工具。

用户可以在完成一段学习后，用一两分钟记录做了什么、投入了多久、属于什么主题、有什么收获；之后通过学习日志和统计回看持续学习留下的轨迹。

产品的核心体验是一条短闭环：

```text
快速记录今天的学习
        ↓
按日期、主题或关键词回找记录
        ↓
看见学习的连续性、投入和主题积累
```

Vibe学习记不是待办清单、番茄钟、课程管理器或社交打卡平台。它不使用完成率、排行榜和强游戏化机制制造压力。

## 2. 目标用户与产品价值

目标用户是正在学习 Vibe Coding、微信小程序、Agent 或其他技术主题的学习者。

产品帮助用户：

- 回答“我今天学了什么、投入了多久”。
- 分别记录同一天发生的多段学习，而不是只留下一个笼统的完成状态。
- 回找上次学习某个主题时做过什么、留下了什么想法。
- 看见连续学习、累计投入和学习主题的变化。

用户不需要注册新的产品账号或填写头像、昵称和个人资料。

## 3. 领域术语

### 学习记录（Learning Record）

一次独立的学习活动，是产品中的最小业务单位。同一天可以有多条学习记录。

### 打卡日（Check-in Day）

至少包含一条学习记录的本地日历日期。同日创建三条学习记录，仍然只算一个打卡日。

### 当前连续（Current Streak）

如果今天有记录，从今天开始向前计算连续打卡日；如果今天没有记录而昨天有记录，则从昨天开始计算；今天和昨天都没有记录时为 0。

### 最长连续（Longest Streak）

全部历史打卡日中最长的一段连续日期天数。

### 学习主题（Tag）

附在学习记录上的可复用主题，例如“微信小程序”“Agent”“GitHub”。一条记录可以没有主题，也可以有多个主题。

## 4. 数据模型与业务规则

### 4.1 学习记录

```ts
type LearningRecord = {
  id: string
  date: string
  createdAt: number
  updatedAt: number

  content: string
  duration: number
  tags: string[]
  takeaway?: string
}
```

字段规则：

| 字段 | 规则 |
| --- | --- |
| `id` | 每条记录唯一，创建后不可修改。 |
| `date` | 创建设备当天的本地日期，格式为 `YYYY-MM-DD`；创建后不可修改。 |
| `createdAt` | 创建时间戳，创建后不可修改。 |
| `updatedAt` | 创建时与 `createdAt` 相同；每次编辑成功后更新。 |
| `content` | 必填，去除首尾空白后为 1–300 个字符；保留正文中的换行。 |
| `duration` | 必填，5–600 分钟，以 5 分钟为步长。 |
| `tags` | 选填，最多 3 个；每个去除首尾空白后为 1–12 个字符；同一记录内不允许重复。 |
| `takeaway` | 选填，去除首尾空白后最多 140 个字符；空内容不保存。 |

核心规则：

- 新建记录自动使用当天日期，不支持补记过去日期。
- 编辑时不能修改记录日期和创建时间。
- 用户可以在同一天创建任意多条记录。
- 保存期间不能重复提交。
- 删除采用不可恢复的硬删除，并且必须先二次确认。
- 所有日期分组、连续计算和日历标记均使用记录的本地 `date`，不能从 UTC 时间戳重新推导日期。

### 4.2 学习偏好

```ts
type LearningPreference = {
  defaultDuration: number
}
```

- 默认学习时长初始为 30 分钟。
- 可选范围为 5–600 分钟，以 5 分钟为步长。
- 修改后只影响以后新建的记录，不回写历史记录。
- 本版本只承诺学习记录跨设备同步，不承诺学习偏好跨设备同步。

## 5. 用户身份、数据归属与同步

Vibe学习记使用微信身份区分用户，并通过微信云开发保存学习记录。

- 每个用户只能读取、创建、修改和删除自己的学习记录。
- 同一微信账号在不同设备上进入小程序时读取同一份云端记录。
- 学习记录以云端数据为准；重新同步只重新读取云端数据，不用本地数据覆盖云端。
- 页面需要区分同步中、同步成功和同步失败。
- 读取或写入失败时不能显示虚假的成功状态，也不能把失败误判成“没有记录”。
- 创建或编辑失败后保留用户已经填写的内容，并提供重试机会。
- “删除我的全部学习记录”只删除当前微信身份下的学习记录，不表示注销微信账号或产品账号。

## 6. 信息架构与导航

小程序包含三个常驻 Tab 和三个子页面。

```text
Tab
├─ Today
├─ 学习日志
└─ 学习统计

子页面
├─ 记录编辑（新建／编辑共用）
├─ 学习详情
└─ 设置
```

主要导航关系：

```text
Today
├─ 设置入口 → 设置
├─ 记录学习 → 记录编辑（新建）
└─ 今日记录 → 学习详情
                 └─ 编辑 → 记录编辑（编辑）

学习日志
├─ 日期／主题／关键词筛选
├─ 学习记录 → 学习详情
└─ 新增按钮 → 记录编辑（新建）

学习统计
├─ 月历日期 → 学习日志（携带日期条件）
└─ 主题排行 → 学习日志（携带主题条件）
```

导航规则：

- 新建、编辑或删除成功后，返回来源页面并重新读取数据。
- 页面不能依赖另一个页面仍保留在内存里的旧列表。
- 统计页通过路由参数把 `date` 或 `tag` 条件交给日志页。
- 切换 Tab、从子页面返回或重新进入小程序时，当前页面与 Tab 高亮必须一致。

## 7. 页面与交互

### 7.1 Today

Today 让用户打开小程序后立即知道今天是否开始学习、已经投入多少，以及下一步可以做什么。

页面包含：

1. 标题、本地日期和设置入口。
2. 当前连续、今日学习分钟和今日记录数量。
3. 包含今天在内的滚动 7 天日期条；有记录的日期显示标记，只用于反馈，不承担筛选和跳转。
4. 根据今日状态变化的主行动卡。
5. 按创建时间倒序排列的今日记录列表。

主要状态：

| 状态 | 页面表现 |
| --- | --- |
| 没有任何记录 | 数字均为 0；显示“今天还没有开始学习”和“记录第一次学习”。 |
| 有历史记录、今天没有记录 | 当前连续按规则从昨天计算；显示“记录一次学习”。 |
| 今天已有记录 | 显示今日分钟和记录数量；显示“再记录一次学习”和今日记录列表。 |

今日记录卡显示创建时间、时长、内容摘要和主题，点击进入学习详情。内容摘要最多两行。

### 7.2 记录编辑

记录编辑页同时承担新建和编辑。新建模式标题为“记录学习”，编辑模式标题为“编辑记录”。

页面包含：

1. 只读日期提示。
2. 必填的学习内容和字数反馈。
3. 以 5 分钟为步长的学习时长。
4. 已有主题选择和新主题输入。
5. 可选的今日收获。
6. 字段错误、保存失败等反馈。
7. 固定在底部安全区上方的保存按钮。

交互规则：

- 输入不合法时不保存，错误显示在对应字段附近，已输入内容保留。
- 保存中按钮禁用并显示进行中状态。
- 返回时存在未保存修改，需要询问是否放弃。
- 新建成功后回到来源页面；编辑成功后回到学习详情。
- 编辑模式在表单末尾提供“删除此记录”，不与保存按钮并列。
- 删除必须说明影响并进行二次确认。

页面不支持图片、附件、富文本、自动主题推荐、学习计时或补记日期。

### 7.3 学习日志

学习日志用于浏览历史、组合筛选和回找学习上下文。

页面包含：

1. 全部记录的打卡日、记录数量和分钟数摘要。
2. 包含今天在内的滚动 7 天快捷日期条。
3. 搜索学习内容或今日收获的关键词输入。
4. “全部”和已有学习主题组成的横向筛选条。
5. 当前筛选结果的记录数量和分钟数。
6. 按日期倒序组织的时间线。
7. 进入新建记录的悬浮按钮。

筛选规则：

```text
日期条件
AND 学习主题条件
AND 关键词条件
        ↓
结果列表、记录数量和分钟数同时更新
```

- 默认不选日期和主题，展示全部历史记录。
- 日期快捷条每次只能选择一天；再次点击已选日期会取消日期条件。
- 从统计月历进入的日期可以超出快捷日期条范围，日志页仍需显示、应用并允许清除该日期条件。
- 主题每次只能选择一个；“全部”表示没有主题条件。
- 关键词匹配学习内容和今日收获，忽略首尾空白。
- 无匹配结果时显示“没有找到匹配的学习记录”和“清除全部筛选”。

每个日期组显示日期、当天总分钟和记录数量；日期组按日期倒序，组内按创建时间倒序。记录卡显示时间、时长、内容摘要和主题，点击进入学习详情。

### 7.4 学习详情

学习详情用于完整阅读一条记录。

页面包含：

1. 完整日期、星期、创建时间和学习时长。
2. 学习主题；没有主题时不显示该区域。
3. 保留换行且不截断的学习内容。
4. 今日收获；未填写时不显示该区域。
5. 右上角编辑入口。

找不到记录时显示“这条学习记录已不存在或已被删除”，并提供返回学习日志的行动。读取失败时显示不同的错误说明和“重新加载”。

详情页不直接提供删除操作，也不包含评论、分享、收藏、AI 总结或上一条／下一条导航。

### 7.5 学习统计

学习统计帮助用户感知坚持、投入节奏和学习主题。

页面包含：

1. 当前连续、鼓励文字和最长连续。
2. 累计打卡日和累计学习分钟。
3. 当前本地月份的学习日历；有记录的日期显示标记，点击日期进入带日期条件的学习日志。
4. 包含今天在内的滚动 7 天投入趋势；每天显示日期和分钟数，没有记录时为 0。
5. 学习主题 Top 3；按包含该主题的记录数量排序，点击进入带主题条件的学习日志。

一条带有多个主题的记录会分别为每个主题贡献一次计数。同一主题次数相同时，最近使用的主题排在前面。

没有任何记录时只显示空状态和“去记录第一次学习”，不显示空图表、虚构数据或奖励元素。

### 7.6 设置

设置页用于管理学习偏好、同步反馈和用户数据，不是个人主页。

页面包含：

1. 学习偏好：默认学习时长。
2. 数据与同步：云端同步状态、上次同步时间和重新同步。
3. 数据与隐私：删除我的全部学习记录。
4. 关于：产品版本和隐私说明入口。

设置页不包含头像、昵称、个人简介、主题颜色、字体大小、主题管理或提醒设置。

## 8. 统计口径

| 指标 | 计算规则 |
| --- | --- |
| 今日记录数量 | `date` 为今天的学习记录数量。 |
| 今日学习分钟 | `date` 为今天的学习记录 `duration` 总和。 |
| 累计打卡日 | 全部记录中去重后的 `date` 数量。 |
| 累计学习分钟 | 全部记录的 `duration` 总和。 |
| 当前连续 | 今天有记录则从今天开始；否则从昨天开始；向前连续计算打卡日。 |
| 最长连续 | 全部打卡日中最长连续日期段的天数。 |
| 滚动 7 天投入 | 从今天向前共 7 个本地日期的每日学习分钟；无记录为 0。 |
| 主题排行 | 按包含该主题的记录数量降序；次数相同按最近使用时间倒序。 |

同一天存在多条记录时，它们会分别计入记录数量和学习分钟，但该日期在累计打卡日与连续计算中只出现一次。

## 9. 产品状态与反馈

所有读取数据的页面均需区分以下状态：

| 状态 | 要求 |
| --- | --- |
| 加载中 | 在对应内容区域显示加载状态，不把尚未完成的读取当作空数据。 |
| 成功且有数据 | 展示真实记录与派生结果。 |
| 成功但无数据 | 展示与页面目标对应的空状态和一个明确行动。 |
| 失败 | 说明失败，保留仍然可信的内容，并提供重试行动。 |

写操作需要区分等待、成功和失败。成功状态必须来自实际完成的云端操作；失败时保留用户输入或原有数据，不提前更新成无法恢复的假成功状态。

视觉规则、颜色、排版、深色模式和可访问性要求以 [Vibe学习记 UI Design](./ui-foundation-design.md) 为准。

## 10. v1 范围外

- 打卡提醒、订阅消息与通知设置。
- 补记过去日期。
- 学习目标、完成率和复杂习惯排期。
- 主题重命名、合并和批量管理。
- 图片、附件、富文本和数据导出。
- AI 自动分类、学习总结或学习建议。
- 同学协作、排行榜、好友和社交能力。
- 专注计时器及其前后台恢复。
- 年度热力图、复杂时间范围和自定义报表。
